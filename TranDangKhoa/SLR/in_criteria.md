# Tiêu chí Tuyển chọn và Loại trừ (IC/EC Criteria)

Bộ tiêu chí YES/NO dùng để quyết định giữ lại hay loại bỏ các bài báo trong nghiên cứu Systematic Literature Review (SLR).

---

# 1. Inclusion Criteria (IC)

Để được giữ lại, bài nghiên cứu phải thỏa mãn **TẤT CẢ** các tiêu chí sau:

| ID  | Tiêu chí |
| --- | --- |
| IC1 | Bài nghiên cứu được viết bằng tiếng Anh |
| IC2 | Được xuất bản từ năm 2018 trở về sau nhằm đảm bảo tính cập nhật công nghệ |
| IC3 | Được công bố trên các nguồn có phản biện khoa học (peer-reviewed) như journal, conference thuộc IEEE, ACM, Springer, ScienceDirect hoặc các preprint uy tín |
| IC4 | Liên quan trực tiếp đến Automated Test Generation, BDD/Gherkin hoặc ứng dụng LLM trong Software Engineering |
| IC5 | Có kết quả thực nghiệm rõ ràng như số liệu đánh giá, biểu đồ so sánh hoặc metrics cụ thể |

---

# 2. Exclusion Criteria (EC)

Bài nghiên cứu sẽ bị loại nếu thỏa mãn **BẤT KỲ** tiêu chí nào dưới đây:

| ID  | Tiêu chí |
| --- | --- |
| EC1 | Bài nghiên cứu bị trùng lặp (duplicate) với các bài báo đã được thu thập trước đó |
| EC2 | Không tìm được toàn văn tài liệu (PDF/full-text) |
| EC3 | Là poster, extended abstract hoặc bài báo ngắn dưới 4 trang |
| EC4 | Chỉ tập trung vào test execution hoặc test maintenance mà không đề cập đến automated test generation |
| EC5 | Không liên quan đến BDD, Gherkin, LLM hoặc acceptance test automation |
