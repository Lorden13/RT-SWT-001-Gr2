# Nghiên cứu thực nghiệm — Tinh chỉnh câu hỏi nghiên cứu (RQ)

## 1. Câu hỏi nghiên cứu và khung PICO ban đầu

### Câu hỏi nghiên cứu (RQ) ban đầu
*Đối với user stories viết theo format Connextra (P), LLM tự động sinh Gherkin scenarios + step definitions (I) so với BDD viết thủ công (C), có đạt semantic similarity >= 0.85 và executable không lỗi cú pháp không (O)?*

### Khung PICO ban đầu
*   **P (Population):** User stories Connextra format ("As a ... I want... So that...")
*   **I (Intervention):** GPT-4o zero-shot sinh Gherkin scenarios + step definitions
*   **C (Comparison):** BDD scenarios viết tay bởi expert (chuyên gia)
*   **O (Outcome):** Metric 1: Semantic similarity >= 0.85; Metric 2: Executable syntax rate (không lỗi parse)

## 3. Câu hỏi nghiên cứu và PICO chính thức (đã tinh chỉnh)

### Câu hỏi nghiên cứu (RQ) chính thức
Does GPT-4o zero-shot (temperature=0) generate Gherkin acceptance test scenarios that achieve cosine semantic similarity >= 0.85 with expert-written scenarios AND executable syntax rate >= 80% when applied to Connextra-format user stories from SE projects?

### Khung PICO chính thức
*   **P (Population):** User stories Connextra format từ >= 3 dự án phần mềm (SE projects)
*   **I (Intervention):** GPT-4o zero-shot (temperature=0) sinh kịch bản Gherkin và mã step definitions
*   **C (Comparison):** Kịch bản Gherkin do chuyên gia kiểm thử phần mềm (expert software tester) viết tay cho cùng tập User Stories
*   **O (Outcome):**
    *   Metric 1: Cosine semantic similarity (sử dụng mô hình nhúng all-MiniLM-L6-v2) >= 0.85
    *   Metric 2: Executable syntax rate (tỷ lệ không lỗi cú pháp tĩnh khi parse) >= 80%
