| 2 | Using Large Language Models to Generate JUnit Tests: An Empirical Study (2024) | Codex; GPT-3.5-Turbo; StarCoder | HumanEval and EvoSuite SF110 Java benchmarks | Compilation rate; correctness; coverage; test smells | Codex achieved above 80% coverage on HumanEval, but models performed poorly on EvoSuite SF110. | Generated Java tests still contain compilation issues and test smells. |

| 5 | Effective Test Generation Using Pre-trained Large Language Models and Mutation Testing (2024) | MuTAP; pre-trained LLM + mutation testing | Python programs / mutation-testing benchmark | Mutation score; fault revealing capability; coverage | Uses mutation testing to optimize generated tests for fault detection rather than only coverage. | Cost and scalability may be challenging for large systems. |

| 7 | An Empirical Study of Unit Test Generation with Large Language Models (2024) | Five open-source LLMs | 17 Java projects | Coverage; compilation; prompt effectiveness | Compares open-source LLMs and prompt strategies for Java unit test generation. | Need careful replication because open-source models evolve quickly. |
| 8 | Mutation-Guided LLM-based Test Generation at Meta (2025) | ACH; mutation-guided LLM test generation | 10,795 Android Kotlin classes in Meta systems | Mutant detection; engineer acceptance; precision/recall | ACH generated 571 privacy-hardening test cases; engineers accepted 73% of generated tests. | Industrial context may not be fully reproducible outside Meta. |
| 9 | A System for Automated Unit Test Generation using Large Language Models (2025) | AgoneTest | Java projects / complex class-level tests | Coverage; test-suite complexity; assessment workflow | Presents an automated system for generating and assessing complex class-level test suites. | Workshop/system paper; detailed benchmark comparison may be narrower. |
| 10 | Evaluating LLM-Based Test Generation Under Software Evolution (2026) | Eight LLMs under mutation-driven evolution framework | 22,374 program variants | Line coverage; branch coverage; pass rate under SAC/SPC changes | Baseline tests reach 79% line and 76% branch coverage, but performance drops under code evolution. | Recent preprint; needs peer-reviewed confirmation. |
| 11 | LLM-based Mockless Unit Test Generation for Java (2026) | MocklessTester | Defects4J and Deps4J | Line coverage; branch coverage; mutation score; token/time cost | Improves coverage and mutation score by using context-enriched generation and constraint-enforced fixing. | Higher token and time costs than baseline. |
| 12 | Enhancing Automated Unit Test Generation with Large Language Models (2025) | Review of LLM-based unit testing techniques | Studies on test generation, assertion generation, and test evolution | Task coverage; adaptation strategy; evaluation patterns | Summarizes how LLMs are adapted across unit testing tasks. | Review evidence depends on quality and coverage of included studies. |
| 13 | On the Evaluation of Large Language Models in Unit Test Generation (2024) | Multiple LLMs | Unit test generation benchmarks | Evaluation validity; coverage; correctness | Highlights that evaluation protocol strongly influences conclusions about LLM-generated unit tests. | Exact metrics must be checked against full paper before final submission. |
| 14 | Evaluating and Improving ChatGPT for Unit Test Generation (2024) | ChatGPT | Unit test generation benchmark projects | Compilation; coverage; repair improvement | Studies how prompt and repair strategies affect ChatGPT unit-test performance. | Bibliographic details should be verified from the final proceedings. |
| 15 | Enhancing LLM-based Test Generation for Hard-to-Cover Branches via Program Analysis (2024) | LLM + program analysis | Hard-to-cover branch benchmark | Branch coverage; targeted branch success | Combines program analysis with LLM prompting to address hard-to-cover branches. | Preprint status and benchmark details need verification. |
| 16 | LLM-Powered Test Case Generation for Detecting Tricky Bugs (2024) | LLM-powered bug-oriented test generation | Bug detection benchmark | Bug detection; fault-revealing tests; coverage | Focuses on generating tests that expose tricky bugs instead of merely increasing coverage. | May be less comparable to coverage-oriented studies. |
| 17 | A Large-scale Empirical Study on Fine-tuning Large Language Models for Unit Testing (2024) | Fine-tuned LLMs | Unit testing datasets across tasks | Test generation; assertion generation; test evolution | Examines whether fine-tuning improves multiple unit testing tasks. | Needs final bibliographic verification before citation. |
| 18 | ChatUniTest Core and Toolchain for Automated Unit Testing (2024) | ChatUniTest Core; Toolchain | Integrated developer workflow | Tool integration; validation; repair workflow | Provides practical implementation evidence for LLM-based unit-test workflow. | Artifact evidence should be cited together with the associated paper. |
