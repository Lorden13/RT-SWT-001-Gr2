# Inclusion / Exclusion Criteria

## Inclusion Criteria

| Code | Criteria |
|---|---|
| IC1 | The paper is written in English. |
| IC2 | The paper was published between 2018 and 2026. |
| IC3 | The study focuses on LLMs, code LLMs, or generative AI for automated unit test generation. |
| IC4 | The target output includes unit tests, executable test suites, test assertions, or tests generated from source-code/context prompts. |
| IC5 | The paper reports empirical evidence such as coverage, compilation rate, pass rate, mutation score, bug detection, or repair success. |
| IC6 | The study provides enough information to extract model/tool, dataset/context, metric, result, and limitation. |

---

## Exclusion Criteria

| Code | Criteria |
|---|---|
| EC1 | Duplicate record. |
| EC2 | Not directly related to LLM-based unit test generation. |
| EC3 | Focuses only on general code generation, program repair, defect prediction, or documentation. |
| EC4 | Full text is unavailable or too incomplete for extraction. |
| EC5 | No clear empirical metric, dataset, or benchmark is reported. |
| EC6 | Outside the scope, such as CI/CD, security fuzzing, UI testing only, BDD/Gherkin only, or test maintenance without generation. |

---

## Screening Strategy

- **V1 Screening:** Title, abstract, and keywords.
- **V2 Screening:** Method, experiment/evaluation, results, and limitations.
- **Data Extraction:** Only studies included after V2 were used for the evidence table and gap analysis.
